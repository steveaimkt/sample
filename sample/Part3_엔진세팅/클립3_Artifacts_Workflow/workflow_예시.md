# Artifacts & Workflow 예시

## Artifacts란?
에이전트가 만든 **결과물**(코드·문서·이미지 등)을 확인·수정·승인하는 패널.

## Workflow란?
여러 에이전트 작업을 **순서대로 연결**하는 파이프라인 설계.

---

## 이커머스 업무 Workflow 예시

### Workflow 1: 광고 성과 → 키워드 최적화
```
Step 1: input/쿠팡_광고_키워드.csv 분석
Step 2: 저성과 키워드 식별 (ROAS < 100%)
Step 3: 대체 키워드 후보 리스트 생성
Step 4: 키워드별 예상 성과 시뮬레이션
Step 5: 최종 최적화 액션 플랜 → output/키워드_최적화_플랜.md
```

### Workflow 2: 리뷰 분석 → 상세페이지 개선
```
Step 1: input/고객리뷰_500개.csv에서 Pain Point 추출
Step 2: 불만 키워드 → 해결 메시지 매핑
Step 3: 상세페이지 카피에 Pain Point 해소 섹션 추가
Step 4: 개선된 상세페이지 HTML 생성 → output/상세페이지_개선.html
```

### Workflow 3: 경쟁사 분석 → 차별화 전략
```
Step 1: 경쟁사 3곳 상세페이지 & 리뷰 수집
Step 2: 각 경쟁사 강점/약점 비교표 생성
Step 3: 우리 제품 차별화 포인트 도출
Step 4: 차별화 기반 상세페이지 기획안 → output/차별화_기획안.md
```

## Artifacts 검토 체크리스트
- [ ] 수치 근거가 포함되어 있는가?
- [ ] 브랜드 톤(친근+전문)이 유지되었는가?
- [ ] 금지 표현(최저가, 가성비)이 없는가?
- [ ] 타겟 고객의 Pain Point가 반영되었는가?
