# 실습 가이드: 기획부터 와이어프레임까지

## 목표
내 상품으로 상세페이지 와이어프레임 1개 완성

---

## 실습 순서

### Phase 1: Pain Point 리서치 (20분)

**할 일**: NotebookLM에 경쟁사 리뷰·상세페이지 소스를 등록하고 Pain Point를 추출합니다.

```
[프롬프트 예시]
NotebookLM "내 상품 리서치" 노트북에서
경쟁사 리뷰 기반 Pain Point TOP 5를 추출하고
각 Pain Point별 공감 키워드 맵을 만들어줘.
```

**산출물**: `output/내상품_pain_point_리서치.md`

---

### Phase 2: Agent Teams Debate (15분)

**할 일**: 마케터 에이전트와 디자이너 에이전트가 상세페이지 전략을 토론합니다.

```
[프롬프트 예시]
Pain Point 리서치 결과를 바탕으로
마케터 관점과 디자이너 관점에서 상세페이지 전략을 토론해줘.

마케터: 전환율 중심으로 카피·구성·CTA 제안
디자이너: 시각적 임팩트 중심으로 레이아웃·컬러·이미지 제안

각 섹션마다 합의안을 도출해줘.
```

**산출물**: `output/내상품_agent_debate_결과.md`

---

### Phase 3: 기획안 JSON 변환 (10분)

**할 일**: 토론 결과를 피그마 레이아웃 플러그인용 JSON으로 변환합니다.

```
[프롬프트 예시]
Agent Teams Debate 결과를 바탕으로
상세페이지 기획안을 JSON 형식으로 변환해줘.

참고: sample/Part5_CreativeOS/클립3_레이아웃플러그인_피그마MCP/기획안_JSON_예시.json
```

**산출물**: `output/내상품_기획안.json`

---

### Phase 4: 와이어프레임 완성 (15분)

**할 일**: JSON을 피그마 레이아웃 플러그인에 입력하여 와이어프레임을 완성합니다.

1. 피그마에서 레이아웃 플러그인 실행
2. `output/내상품_기획안.json` 업로드
3. 자동 생성된 레이아웃 확인 및 미세 조정

**산출물**: 피그마 와이어프레임 파일

---

## 체크리스트

- [ ] Pain Point가 3개 이상 도출되었는가?
- [ ] 각 Pain Point에 해소 메시지가 매핑되었는가?
- [ ] 마케터·디자이너 관점이 모두 반영되었는가?
- [ ] JSON에 모든 섹션(히어로~CTA)이 포함되었는가?
- [ ] 브랜드 톤(친근+전문)이 유지되었는가?
- [ ] 금지 표현(최저가, 가성비)이 없는가?
